# TIR-2026-001 — Threat Intelligence Report

**Title:** Mirai Botnet Infection — Zeek `conn.log` Analysis
**Analyst:** Tasha (DataSec Chronicles)
**Date:** July 13, 2026
**Classification:** Portfolio / Public
**Source data:** Public Zeek/Corelight `conn.log` (Kaggle: DNS and Connections Log), 1,319,960 records
**ATT&CK version mapped:** v19.1 (April 2026)

---

## 1. Executive Summary

Analysis of a Zeek `conn.log` using SQL identified an active Mirai botnet infection involving multiple compromised internal hosts, an external SSH intrusion, and command-and-control (C2) beaconing to four external servers. A key finding from timestamp analysis: internal reconnaissance and C2 beaconing were active approximately 28 hours **before** the observed SSH intrusion, indicating the network was already compromised when the capture began.

All indicators and technique mappings in this report were verified by direct query against the source data. Where a behavior is observed but its intent cannot be confirmed from logs alone, it is labeled as such.

## 4. MITRE ATT&CK Mapping

*Mapped against ATT&CK Enterprise v19.1.*

| Tactic | Technique | ID | Evidence in this incident |
|---|---|---|---|
| Reconnaissance | Active Scanning | T1595 | ICMP ping sweeps and mass external scanning from .43 and .50 |
| Discovery | Network Service Discovery | T1046 | .50 probing HTTPS/SMB/RPC/FTP across 771 internal hosts |
| Discovery | Remote System Discovery | T1018 | ICMP host-discovery sweeps identifying live internal systems |
| Credential Access | Brute Force: Password Guessing | T1110.001 | 5.45.85.158 and others repeatedly attempting SSH logins |
| Initial Access | External Remote Services | T1133 | External SSH access into the internal network |
| Initial Access | Valid Accounts | T1078 | 3h18m interactive SSH session on .43 (successful authenticated access) |
| Command & Control | Application Layer Protocol: Web Protocols | T1071.001 | Beaconing to external C2 (protocol likely web-family; see note) |
| Command & Control | Non-Standard Port | T1571 | C2 traffic on ports 4431 and 8343, not standard 443/80 |
| Command & Control | Fallback Channels | T1008 | Even distribution across 4 C2 servers and 2 ports — redundancy |
| Impact | Network Denial of Service | T1498 / T1499 | Mirai's operational objective (DDoS); recruitment observed, DDoS not observed in this capture |

**Analyst notes on the mapping:**
- **T1110 is a Credential Access technique, not Initial Access.** Brute force *enables* access but ATT&CK categorizes the guessing itself under Credential Access. The Initial Access techniques here are T1133 (External Remote Services) and T1078 (Valid Accounts).
- **T1071.001 is assessed as likely, not confirmed.** The beacons ran on non-standard ports (4431/8343); a port does not confirm the application protocol. Confirming web protocols would require payload or service-field analysis.
- **T1498/T1499 is the threat's objective, not an observed behavior.** Mirai recruits hosts for DDoS; this capture shows recruitment and C2, not an executed DDoS.

**Mapping verification workflow used for this report:**
1. Verify the ATT&CK version (v19.1) and state it explicitly.
2. Check each technique ID exists in that version.
3. Confirm any referenced sub-techniques still exist.
4. Verify each technique name hasn't changed.
5. Confirm the tactic(s) are still accurate.
6. Ensure every artifact (report, blog, repo) references the same ATT&CK version.

## 5. Attack Timeline (first-seen in capture, UTC)

| Time | Event | Host |
|---|---|---|
| May 1, 12:34 | Internal recon begins | 192.168.10.50 |
| May 1, 17:38 | C2 beaconing begins | 192.168.60.22 |
| May 2, 16:36 | SSH intrusion begins (~28h later) | 5.45.85.158 → .43 |
| May 2, 17:05 | Mirai scanning begins (+29 min) | 192.168.10.43 |

**Assessment:** The SSH intrusion explains the compromise of `.43` (scanning begins 29 minutes after the attacker's first session) but does not account for `.50` and `.60.22`, which were active more than a day earlier. This indicates a prior compromise predating the observed intrusion. *Caveat: these are first-seen-in-capture times, not necessarily true patient-zero — an earlier compromise may predate the logged window.*

## 6. Recommendations

1. **Isolate confirmed compromised hosts** — `.43` (Mirai scanner) and `.60.22` (C2 beaconing) for forensic imaging and remediation.
2. **Block C2 infrastructure** at the perimeter — the four external C2 IPs, and alert on outbound traffic to ports 4431/8343.
3. **Investigate `.50`** — confirm whether the network-wide sweep is a compromised host or an authorized scanner before final classification.
4. **Analyze the DNS log** — cross-reference the four C2 IPs and 5.45.85.158 against `dns.log` to identify domain-based infrastructure.
5. **Hunt for the earlier compromise** — recon and C2 predate the SSH intrusion; earlier logs may reveal the true initial access vector for `.50` and `.60.22`.
6. **Enforce credential hygiene** — the Mirai vector is default/weak credentials on internet-facing services; audit and rotate.

---

## 7. Confidence & Limitations

This assessment is based on `conn.log` connection metadata only. It does not include payload inspection, DNS correlation, endpoint telemetry, or authentication logs. Behavioral classifications (e.g., `.50` as reconnaissance) describe observed traffic patterns; intent is inferred and noted where unconfirmed. All quantitative findings are reproducible against the source dataset via the queries at github.com/Tash925/mirai-zeek-detection.
