01 Executive Summary

Analysis of a Zeek conn.log using SQL identified an active Mirai botnet infection: multiple compromised internal hosts, an external SSH intrusion, and command-and-control beaconing to four external servers.

A key finding from timestamp analysis: internal reconnaissance and C2 beaconing were active roughly 28 hours before the observed SSH intrusion — the network was already compromised when the capture began. All indicators and technique mappings were verified by direct query against the source data. Where a behavior is observed but its intent cannot be confirmed from logs alone, it is labeled as such.

Analyst notes on the mapping
  • T1110 is Credential Access, not Initial Access. Brute force enables access, but ATT&CK categorizes the guessing itself under Credential Access. Initial Access here is T1133 and T1078.
  • T1071.001 is assessed as likely, not confirmed. Beacons ran on non-standard ports (4431/8343); a port does not confirm the application protocol. Confirming web protocols needs payload or service-field analysis.
  • T1498/T1499 is the objective, not an observed behavior. This capture shows recruitment and C2 — not an executed DDoS.
Mapping verification workflow
  • Verify the ATT&CK version (v19.1) and state it explicitly.
  • Check each technique ID exists in that version.
  • Confirm any referenced sub-techniques still exist.
  • Verify each technique name hasn't changed.
  • Confirm the tactic(s) are still accurate.
  • Ensure every artifact references the same ATT&CK version.

This indicates a prior compromise predating the observed intrusion. Caveat: these are first-seen-in-capture times, not necessarily true patient-zero — an earlier compromise may predate the logged window.

06 Recommendations

  1. Isolate confirmed compromised hosts — .43 (scanner) and .60.22 (C2) for forensic imaging and remediation.
  2. Block C2 infrastructure — the four external C2 IPs; alert on outbound traffic to ports 4431/8343.
  3. Investigate .50 — confirm whether the sweep is a compromised host or an authorized scanner before final classification.
  4. Analyze the DNS log — correlate the four C2 IPs and 5.45.85.158 against dns.log to identify domain infrastructure.
  5. Hunt the earlier compromise — recon and C2 predate the SSH intrusion; earlier logs may reveal the true initial-access vector.
  6. Enforce credential hygiene — the Mirai vector is default/weak credentials on internet-facing services; audit and rotate.

07 Confidence & Limitations

This assessment is based on conn.log connection metadata only — no payload inspection, DNS correlation, endpoint telemetry, or authentication logs. Behavioral classifications (e.g. .50 as reconnaissance) describe observed traffic patterns; intent is inferred and noted where unconfirmed. All quantitative findings are reproducible against the source dataset.
